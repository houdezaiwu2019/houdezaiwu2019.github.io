ShuffleNet网络详解与Pytorch实现

本文介绍的是ShuffleNet网络，主要的两点就是channel shuffle操作以及相关的网络设计技巧，以及代码的编写技巧。但是究其本质其实和MobileNet v2一样，都是在原有ResNet基本结构的一种改进。

参考视频：https://www.bilibili.com/video/BV15y4y1Y7SY/?spm_id_from=pageDriver&vd_source=14752a83e7c12513cb40e176641418e8

ShuffleNet v1

概述

首先在ShuffleNet v1里面提出了channel shuffle的思想，而且在Shuffle Unit的block里面的全部使用的是GConv和DWConv。

下面是网络的性能指标。

这里可以看出，其相比MobliNet v1来说有一定的提高，虽然不是特别的明显，但是有一定的参考意义。

  • 普通的组卷积GConv是针对一般的来说的，尽管其可以减少参数计算量，但是其通道之间是不互通的结构。
  • 于是改进之前的组卷积结构，将不太Group细分，然后重新排列，这样使得下一层组卷积可以学到不同组之间的信息。

网络结构

原论文的中通过分析不同层之间的运算量发现，只要的运算量其实是被1x1的卷积层占用了，而不是组卷积层。

  • 因此如(b)为stride=1的分支，其主要普通的1x1的卷积全部换成GConv，并且增加Channel Shuffle层。

  • 如(c)为stride=2的分支，相比之前的ResNet网络的分支，其捷径分支是通过一个3x3的池化下采样层来实现的，其stride=2，对于的主分支上的DWConv也为3x3；

    之后两个分支是通过Concat拼接在一起，而不是相加。

  • 下面是网络的参数表，这里主要关注g=3，原文中很多的实验都是基于这组参数的。

标注的部分主要是说：

中间的编码的层的参数是基于是resnet进行修改的，是一般得瓶颈结构，中间部分参数是两边的1/4，不是MobileNet里面的倒金字塔结。

MobileNet 主要说明倒金字塔结构对应的参数损失比较少。

其次就是Stage2的第1层是不用组卷积的，原因就是channel数比较少的情况下，其效果提升不是很明显。

参数量计算

以上是对应每个block的参数量。

  • 组卷积的参数量是一般参数量的$1/g$；
  • 对应pointwise来说，其对应的组卷积组数是$1/m$.

Flops是理论的计算量，不是实际的效果，仅供参考。

ShuffleNet v2

概述

Flops准确来说其实就是每秒运算浮点运算的次数，针对的硬件来说就是衡量其运算能力的指标，针对算法就是算法的计算量和复杂程度的衡量。

设计准则

  • Flops仅仅是个间接的评价指标，最直接的指标还是其运算的速度。
  • 首先，衡量运算速度的指标除了Flops之外，内存访问的开销 (memory access cost )MAC，其主要是针对组卷积计算。
    • 除此之外，GPU在实际的运算过程中也不仅仅是外卷积服务的，其也会参与到shuffle、Data的操作中，而Flops仅仅是衡量的是卷积的参数个数。
    • 并行度(degree of parallelism)，也是衡量模型也运算快慢的重要的指标，相同Flops下，并行度越高，速度越快。
  • 其次，不同的硬件平台上的效果也是不同的，对应GPU和ARM而言，其内存的消耗也大有差异。
  • 总的来说，Flops是远远不够的。

本文总结了设计出高效的网络的四条建议：

  • G1 : Equal channel width minimizes memory access cost (MAC)

  • G2: Excessive group convolution increases MAC

  • G3: Network fragmentation reduces degree of parallelism

  • G4: Element-wise operations are non-negligible

G1 : Equal channel width minimizes memory access cost (MAC)

  • $c_1,c_2$代表是输入和输出的通道数；
  • 由于均值不等式的取整条件，其在$c_1=c_2$达到最小值，但是前提是Flops即B参数是不变的。

针对上述的准则进项了相关的实验：

  • 实验是将连个卷积堆叠之后，第一个CNN是$c_1 \rightarrow c_2$，第二个CNN是$c_2 \rightarrow c_1$;
  • 可以看到两者的比值越小，即差距越大，其推理分速度就越慢。但是可以看到对于不同的硬件平台，其提升的效果也是不同的。

G2: Excessive group convolution increases MAC

  • $hwc_1$代表的是输入特征矩阵占用内存的情况；
  • $hwc_2$代表的是输出特征矩阵占用内存的情况；
  • $c_1 c_2/g$代表的是卷积占用内存的情况；

这里当Flops不变的情况下，$B/g$ 和 $B/hw$都是定值，于是MAC和g就是正相关，g越大，MAC就越大。

如图为测试的效果，在GPU上测试时的效果比较的明显，之后CPU上测试的效果就不是那么明显，其提升的效果连一般都达不到。

G3: Network fragmentation reduces degree of parallelism

网络设计的碎片化程度越高，速度就越慢。

  • 分支主要包含串联和并联的，分支数量越多，其运算的效果就越差；
  • 对于每个卷积，都有一个启动和同步的过程；针对并行结构，计算快的卷积需要等待计算慢的卷积，效率低。
  • 在GPU上的运算效果是，随着碎片化的结构变大，其推理速度变短。

G4: Element-wise operations are non-negligible

Element-wise操作其实主要是指的是元素级别的操作，主要的包含的是激活函数RELU操作，AddTenor,AddBias操作。

这类操作尽管气间Flops很小，但是其对于的MAC很大，即需要很大的内存开销。

实验的结果说明：去掉的RELU和short-cut之后可以提升20%的推理速度。

于是将上述的四个进行总结。

网络结构

如上图所示，(a)(b)对应是shuffleNet v1步距为1和2的情况，(c)(d)是考虑上述准则之后，设计出来得到shuffleNet v2 步距为1和2的情况。

  • 针对(c)，其网络是设计的是将输入的特征层分成$c’$和$c-c’$两部分，将其中的一部分的输入到捷径分支，另一部分输入到三个channel数不变(为了满足G1)的卷积上；
    • 其中的1x1的卷积不再使用组卷积GConv（为了满足G2准则)，由于其变成普通的卷积，于是将channel shuffle移到最后一层。
    • 之后进行concat操作(为了满足G1)。
  • 其次就是RELU的位置出现了调整，v1是将两个分支的数据相加或concat之后再进行的，v2是仅仅再有卷积操作的分支使用，捷径分支不使用上述操作了。
  • 对于连续的三个element-wise操作Concat, Channel shuffle,Channel split，可以将其合并成一个整体，进而减小操作的个数。(满足G4准则)

  • 针对(d)，下采样操作就不再进行Channel split，因此输出就直接变成原来的两倍

  • 原文给出的网络是随机划分的大小是原来的1/2，即$c’=c/2$。

以上是网络的主要的结构：

  • v1 和 v2的主要框架是相同的，唯一的不同是增加了Conv5 1x1的卷积层。
  • 对于每个Stage的第一个卷积是下采样+channel翻倍，但是Stage2 的第一的channel就是一个特例，因为两个输出channel并不等于输出channel，于是就需要进行channel数的改变，而没有完全遵循G1。

性能指标

  • 可以看出再GPU上ShuffleNet 是没有MoblieNet快，但是其准确率和在ARM上的运行速度是相当快的。
  • 比较的前提是Flops相同。

  • 加入注意力机制之后，其预测的准确率和运算的速度大大超越的先前的模型。
